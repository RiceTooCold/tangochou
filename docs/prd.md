# PRD：日文單字練習應用程式

**文件版本：** v2.0  
**最後更新：** 2026-04-14  
**狀態：** Draft

---

## 1. 背景與目標

### 背景

現有的日語教材（如 NTU 日文課第七～九課的單語表 PDF）提供了完整的單字資訊，包括漢字、假名讀音、聲調標記（⓪①②③④）與中文翻譯。但學習者缺乏有效的互動練習工具，只能靠紙本反覆翻閱。

### 目標

打造一個輕量的日文單字練習 Web App，具備以下核心能力：

1. **以 JSON 儲存結構化單字資料**（預先從 PDF 解析，靜態打包）
2. **提供多種練習模式**讓學習者有效記憶
3. **追蹤學習進度**

### 非目標（Out of Scope）

- 不做用戶帳號系統（無需後端資料庫，localStorage 即可）
- 不做多語言 UI（中文介面即可）
- 不做語音合成（TTS）
- 不做手寫辨識
- 不做多裝置同步
- 不做 runtime PDF 解析（資料預先處理）

**主要使用者：** 修習日語課程的大學生

**典型場景：**

| 場景 | 說明 |

## 3. 資料層設計

### 3.1 資料來源

單字資料預先從 PDF 抽取為 JSON，靜態打包進 bundle。目前涵蓋第 7～12 課，以 vision workflow 產生（每頁子代理看圖抽取 → 文字層字元交叉校對 → 人工複檢），取代舊的 `scripts/parse_vocab.py`（pdfplumber + regex）。

資料檔案位於 `src/data/`：
- `vocab-lesson-7.json`（95 個單字）
- `vocab-lesson-8.json`（60 個單字）
- `vocab-lesson-9.json`（57 個單字）
- `vocab-lesson-10.json`（50 個單字）
- `vocab-lesson-11.json`（74 個單字）
- `vocab-lesson-12.json`（77 個單字）

### 3.2 資料格式

每個單字物件：

```json
{
  "id": "l9-001",
  "word": "座る",
  "reading": "すわる",
  "accent": [0],
  "meaning": "坐",
  "section": "文型１",
  "lesson": 9,
  "tags": ["動詞"]
}
```

**欄位說明：**
- `accent`: 聲調類型陣列 `number[]`，0 = 平板型，1~9 = 各拍降核；可多值表雙聲調（如 `[1,2]`），無標記則 `[]`。第 11/12 課已採陣列；第 7/8/9 課待遷移（目前仍為單一 int）
- `section`: 所屬文型或分組
- `tags`: 可選，詞性標記

### 3.3 本地持久化

`localStorage` 儲存學習進度，key 為 `jp-vocab-progress-lesson-{N}`：

```json
{
  "progress": {
    "l9-001": {
      "status": "known",
      "reviewCount": 3,
      "lastReviewed": "2026-04-14T00:00:00Z"
    }
  }
}
```

`status` 值：`"known"` | `"unknown"` | `"unseen"`

## 4. 功能規格

### 4.1 首頁（HomeScreen）

- 合併顯示所有課程的總進度（認識 N / 共 N 個）
- 可篩選課別（第 7、8、9 課）
- 各練習模式入口（閃卡、選擇題、默寫）
- 單字列表入口

### 4.2 練習模式

#### Mode A：閃卡（Flashcard）

| 步驟 | 說明 |
|------|------|
| 正面 | 顯示日文（含漢字）+ 聲調數字 |
| 翻面 | 顯示假名讀音 + 中文意思 |
| 評分 | 使用者點擊「認識」或「不熟」 |
| 結果 | 進度寫入 localStorage |

**設定選項：**
- 正面語言：日文 → 中文 or 中文 → 日文
- 是否顯示聲調
- 範圍：全部 / 指定文型 / 僅不熟的

#### Mode B：選擇題（Quiz）

- 題目：顯示單字（日文或中文）
- 選項：4 選 1（1 個正確 + 3 個干擾項，從同課隨機取）
- 計時：可選開啟 15 秒倒數
- 統計：結束後顯示正確率、錯誤清單

#### Mode C：默寫（Typing）

**題目：** 顯示中文意思，每題隨機要求輸入漢字或讀音（各約 50%）

- 題目下方標示本題要求，e.g.「請輸入漢字」/ 「請輸入讀音」
- 答題後顯示漢字 + 讀音 + 中文，方便對照
- 適合進階學習者

**Fuzzy 判斷規則（輕量）：**

| 情況 | 處理 |
|------|------|
| 全形／半形混用 | 正規化後比對（e.g. `Ａ` = `A`） |
| 前後空白 | trim 後比對 |
| 讀音長音：`おう` / `おお` 混用 | 視為相同 |
| 漢字題輸入正確假名 | 算對，但顯示「正確！（漢字：座る）」提示 |

### 4.3 單字管理

- 全覽列表：依文型分組，顯示所有單字
- 篩選器：文型 / 熟悉程度（認識 / 不熟 / 未見）
- 搜尋：關鍵字即時過濾（日文、中文皆可）
- 重置進度：清除 localStorage 進度

## 5. 技術架構

### 技術選型

| 層 | 選擇 | 理由 |
|----|------|------|
| 前端框架 | React + Vite（最新版）| 輕量、快速 HMR |
| 樣式 | Tailwind CSS（最新版）| 原子化 CSS，快速開發 |
| 單字資料 | 靜態 JSON，build time import | 無需 API，簡單可靠 |
| 學習進度 | `localStorage` | 輕量，無需帳號系統 |

## 附錄：聲調符號說明

`accent` 以陣列儲存（`number[]`）；同一詞出現多個圈即多值（雙聲調）。

| PDF 標記 | accent 值 | 類型 |
|----------|-----------|------|
| ⓪ | 0 | 平板型 |
| ①②③④ | 1~4 | 各拍降核 |
| ⑤⑥⑦⑧ | 5~8 | 多音節詞降核（長詞，如 おはようございます=⑧、ご馳走様でした=⑦）|
